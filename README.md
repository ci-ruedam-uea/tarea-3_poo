# Comparación entre Programación Tradicional y Programación Orientada a Objetos en Python

## Descripción del proyecto
Este proyecto tiene como finalidad comparar la Programación Tradicional y la Programación
Orientada a Objetos (POO) mediante el desarrollo de un programa en Python que calcula el
promedio de notas de un estudiante.  
Se implementan dos soluciones diferentes para resolver el mismo problema, permitiendo
analizar la organización, estructura y ventajas de cada enfoque.

---

## Objetivo general
Desarrollar habilidades prácticas en Python mediante la implementación de un programa
que permita calcular el promedio de notas de un estudiante, aplicando la Programación
Tradicional y la Programación Orientada a Objetos.

---

## Programación Tradicional
En este enfoque el programa se organiza a través de funciones independientes.
Cada función cumple una tarea específica, como ingresar las notas y calcular el promedio.
Este tipo de programación es sencilla de entender y adecuada para programas pequeños,
aunque puede resultar difícil de mantener cuando el proyecto crece.

Archivo:
- `códgio tradiconal.py`

---

## Programación Orientada a Objetos (POO)
En este enfoque se utiliza una estructura basada en clases y objetos.  
Se aplican los principales pilares de la POO:

- **Abstracción:** mediante una clase base que define el comportamiento general.
- **Encapsulación:** protegiendo los datos internos del estudiante.
- **Herencia:** reutilizando características de una clase base.
- **Polimorfismo:** sobrescribiendo métodos para adaptarlos a la clase estudiante.

Este enfoque permite una mejor organización del código y facilita su mantenimiento y
ampliación.

Archivo:
- `código poo.py`

---

## Comparación de enfoques
Ambos enfoques permiten resolver el mismo problema, sin embargo:

- La Programación Tradicional es más simple y directa.
- La Programación Orientada a Objetos ofrece una estructura más ordenada, reutilizable
  y escalable.

Por esta razón, la POO es más recomendable para proyectos de mayor tamaño o complejidad.
